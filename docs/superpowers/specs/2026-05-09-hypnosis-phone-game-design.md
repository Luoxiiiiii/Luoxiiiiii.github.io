# 催眠主题手机解谜游戏 —— 设计文档

## 概述

一款以手机界面为核心、叙事驱动的网页解谜游戏。玩家拿到姐姐的手机，通过翻阅她的聊天记录、浏览器历史、相册、备忘录等数字痕迹，调查一个神秘深夜电台背后的催眠阴谋，最终发现自己也早已被卷入其中。

纯前端网页，零服务器，适合小范围分享。

---

## 核心设定

| 要素 | 内容 |
|------|------|
| 主角 | "你"——姐姐的妹妹 |
| 被调查者 | 姐姐——最近沉默寡言、深夜不睡、眼神空洞 |
| 触发事件 | 你拿到了姐姐的手机，发现她最近行为异常 |
| 核心谜团 | 一个只在深夜有内容的奇怪电台（87.9 MHz） |
| 玩法载体 | 模拟手机界面，自由翻阅各个app |
| 时间系统 | 白天/深夜手动切换，特定内容锁定在某时段 |
| 核心工具 | "窥探器"app——关键词搜索和线索关联 |
| 结局 | 双重嵌套反转：剧情反转 + 元叙事界面破墙 |

## 游戏结构

### 三幕流程（总时长约45-90分钟）

**第一幕 · 发现（15-25分钟）**
- 玩家打开手机，自由浏览各个app
- 在消息中看到姐姐和未知号码的奇怪对话
- 浏览器搜索历史指向一个电台
- 第一次切换到深夜 → 电台app传出内容
- 初步确定"这个电台有问题"

**第二幕 · 调查（20-40分钟）**
- 白天/深夜切换解锁不同内容
- 窥探器成为核心工具——跨app关键词搜索
- 发现更多联系人、网页、隐藏记录
- 拼凑出电台是一个催眠诱导工具的真相
- 发现姐姐不是唯一受害者

**第三幕 · 反转（10-20分钟）**
- 窥探器解锁最终线索
- 剧情反转：主角发现自己也是被催眠者
- 元反转：手机屏幕出现异常——闪烁、扭曲、催眠暗示浮现
- 尾声："现在你也是听众了。晚安。"

### 时间系统
- 手动切换白天/深夜，无真实时间等待
- 部分内容仅在某时段可见
- 切换会触发新消息/新内容"到达"

## 手机界面设计

### 布局规格
- 比例 320×680px，圆角机身 + 刘海
- 深色背景，模拟真实手机屏占比
- 状态栏（时间、信号、电量）

### App列表

| App | 用途 | 关键内容 |
|-----|------|----------|
| iMessage | 姐姐与多人的聊天记录 | 未知号码、闺蜜、妈妈、神秘人X |
| 浏览器 | 搜索历史、书签、标签页 | 电台论坛、催眠相关搜索 |
| 电台 | 调频收音机界面 | 87.9 MHz 深夜内容 |
| 电话 | 通话记录 | 深夜拨打未知号码 |
| 相册 | 照片、截屏 | 室内日记、模糊夜间照片 |
| 备忘录 | 零散笔记 | 催眠后意识流文字、密码提示 |
| 邮件 | 邮件收件箱 | 可疑邮件 |
| 窥探器 | 关键词搜索+线索关联（专属工具） | 解锁隐藏内容 |

## 谜题体系

### 谜题类型（由浅到深）

1. **跨app密码发现** —— 在A处找到密码，解锁B处的锁
   - 例：聊天里提到密码线索 → 浏览器搜索 → 解锁备忘录

2. **窥探器关键词搜索** —— 输入发现的关键词，建立跨app关联
   - 例：搜索"87.9" → 显示所有相关的聊天、浏览记录、备忘录

3. **照片线索识别** —— 相册照片中包含隐藏信息
   - 例：模糊的窗外照片中有一块发光路牌

4. **社交选择** —— 假装姐姐回复消息，选择回复内容影响信息获取
   - 例：神秘人问"听了吗" → 不同回复导致不同结果

5. **简单编码/密码破译** —— 备忘录中的数字/符号需要解读
   - 例：一组数字需要联想转换成日期或频率

### 设计原则
- 无失败状态——所有谜题可凭坚持解开
- 每个谜题3-5分钟解决
- 一个谜题的解法跨越2-3个app
- 谜题伪装成正常的手机操作

## 结局实现

### 剧情反转机制
- 最后一条线索揭示：主角接触这部手机的动机也是被植入的
- 暗示主角在"调查"的过程中，同样在收听/接触催眠内容
- 电台内容在后期逐渐变得"像是在对玩家说话"

### 元叙事实现（前端）
- 界面开始出现闪烁、像素扭曲
- CSS动画实现螺旋/波纹图案叠加
- 文字出现重复、错位
- 最后画面定格在催眠暗示语

## 资产方案

| 资产类型 | 方案 |
|----------|------|
| 相册图片 | AI生成（Midjourney / DALL-E 3） |
| app图标 | CSS绘制，无需图片 |
| 聊天背景 | CSS实现 |
| 网页截图 | HTML模拟 |
| 电台音频 | 改为文字展示（"正在播放..." + 逐行文本） |
| 音效 | 无（纯文字/视觉叙事） |

---

## 技术要求

- 纯前端 HTML + CSS + JavaScript
- 无框架依赖（单页面应用）
- localStorage 用于进度保存
- 静态托管，支持 GitHub Pages / Netlify
- 所有音频为静态文件播放（如后期添加）

---

## 内容规模参考

- 消息记录：3-4个联系人，每个5-15条消息
- 浏览器页面：3-5个模拟网页
- 电台内容：3-5段夜间"广播"文本
- 相册照片：5-10张AI生成图片
- 备忘录：3-5篇笔记
- 谜题：3-4个轻量谜题
- 游戏总时长：45-90分钟
